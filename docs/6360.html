一、 目的

本程序書旨在規範資訊系統建置、維護、稽核及管控作業流程，確保資訊系統安全、穩定、有效運作，並符合相關法規及公司政策。

二、 範圍

本程序書適用於公司所有資訊系統的建置、維護、稽核及管控，涵蓋但不限於：

  • 資訊系統規劃與設計
  • 軟體開發與測試
  • 硬體採購與安裝
  • 系統佈署與移轉
  • 系統維護與更新
  • 系統安全管理
  • 系統稽核與查核
  • 緊急事件處理

| 角色 | 責任 | 權限 |

四、 作業流程

(一) 資訊系統建置流程:

1. 需求分析: 收集使用者需求，進行可行性分析，撰寫需求規格說明書。
2. 系統設計: 根據需求規格說明書，設計系統架構、資料庫、介面等。
3. 開發測試: 進行程式開發、單元測試、整合測試、系統測試。
4. 佈署上線: 將系統佈署至生產環境，進行資料移轉及驗收測試。
5. 文件編製: 編寫系統操作手冊、維護手冊等相關文件。

(二) 資訊系統維護流程:

1. 例行維護: 定期進行系統備份、效能監控、安全檢查等例行維護工作。
2. 問題處理: 針對系統異常狀況，進行問題診斷、修復及追蹤。
3. 系統更新: 定期更新系統軟體、修補安全漏洞。
4. 系統升級: 根據需求，進行系統升級或擴充。

(三) 資訊系統稽核流程:

1. 稽核計畫: 制定年度稽核計畫，明確稽核範圍、目標、方法及時間表。
2. 稽核執行: 根據稽核計畫，執行系統稽核，收集相關證據。
3. 稽核報告: 撰寫稽核報告，說明稽核結果、發現的問題及建議改善方案。
4. 改善追蹤: 追蹤改善方案的執行情況，確保問題得到有效解決。

(四) 緊急事件處理流程:

1. 事件通報: 發生緊急事件時，立即向相關人員通報。
2. 事件處理: 根據緊急事件處理計畫，執行事件處理流程。
3. 事件記錄: 詳細記錄事件發生過程、處理方法及結果。
4. 事後檢討: 事件處理後，進行事後檢討，找出問題根源，制定預防措施。
