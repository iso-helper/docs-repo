4. 程序

4.1 文件的控制

4.1.1 文件的創建: 文件應由經授權的人員創建，內容應清晰、準確、完整且易於理解。創建文件時需明確文件編號、修訂版號、發佈日期、作者及批准者。
4.1.2 文件的審閱與批准: 所有文件在發佈前必須經過審閱和批准。審閱應確認文件符合相關標準及公司要求。批准者應明確標示其身份及批准日期。
 4.1.3 文件的分發: 已批准的文件應分發給所有相關人員。分發方式應確保文件能有效傳達至需知悉人員，並應建立分發記錄。
4.1.4 文件的修訂: 文件修訂應以文件控制程序進行，包括新版號的標識、修訂內容的標示、修訂日期及修訂原因。舊版文件應作廢並妥善保管。
 4.1.5 文件的變更控制: 所有文件變更應追蹤記錄，並應明確顯示變更內容、變更日期及變更者。
4.1.6 文件的儲存: 文件應儲存在安全、易於存取且受控的環境中，防止損壞、遺失或未經授權的存取。
 4.1.7 文件的處置: 作廢文件應明確標示作廢狀態，並按規定程序妥善處置，避免誤用。處置方法應符合公司相關規定及法律法規。

4.2 記錄的控制

4.2.1 記錄的創建: 記錄應在活動發生時或盡快創建，內容應清晰、準確、完整及可追溯。記錄應明確標識記錄編號、創建日期、相關活動及人員。
 4.2.2 記錄的審閱: 記錄應由相關人員進行審閱，確保其準確性和完整性。
4.2.3 記錄的儲存: 記錄應儲存在安全、易於存取且受控的環境中，防止損壞、遺失或未經授權的存取。儲存期限應符合公司規定及法規要求。
 4.2.4 記錄的處置: 記錄應根據規定的保留期限進行保管，逾期應按規定程序妥善處置。處置方法應符合公司相關規定及法律法規。
