1. 目的

本程序書旨在規範本組織（機構名稱）之資訊系統存取控制的管理作業，確保資訊系統的機密性、完整性及可用性，並符合相關法規及政策的要求。

2. 適用範圍

本程序書適用於本組織所有資訊系統及相關資源的存取控制管理，包含但不限於：伺服器、工作站、應用程式、資料庫、網路設備等。

3. 職責

  • 資訊安全主管: 負責制定及修訂本程序書，監督執行情況，並處理相關異常事件。
  • 系統管理員: 負責系統存取權限的設定、維護及稽核，並根據本程序書執行相關作業。
  • 使用者: 負責遵守本程序書規範，並妥善保管其帳號密碼。

4. 作業流程

4.1 帳號申請與核准:

1. 使用者提出帳號申請，填寫「帳號申請表」（附件一），並經主管核准。
2. 系統管理員根據申請表建立帳號，並賦予使用者相應的存取權限。
3. 系統管理員將帳號資訊通知使用者，並指導使用者更改初始密碼。

4.2 存取權限管理:

1. 存取權限應基於最小權限原則，只授予使用者執行工作所需的最少權限。
2. 系統管理員定期檢視並更新使用者的存取權限，移除不再需要的權限。
3. 存取權限的變更應記錄在「存取權限變更記錄表」（附件二）。

4.3 密碼管理:

1. 使用者應設定強健的密碼，並定期更改密碼（例如，每90天）。
2. 密碼不得洩露給他人，且應妥善保管。
3. 系統應具有密碼複雜度檢查機制。
4. 針對密碼遺失或忘記的情況，應有明確的處理流程（參考附件三）。

4.4 帳號停用與刪除:

1. 使用者離職或轉職時，其帳號應立即停用或刪除。
2. 帳號停用或刪除應記錄在「帳號停用/刪除記錄表」（附件四）。

4.5 存取稽核:

1. 系統應定期進行存取稽核，記錄所有使用者的存取行為。
2. 系統管理員應定期檢視稽核記錄，並追蹤異常活動。
3. 稽核記錄應妥善保存，並符合法規要求。

4.6 異常事件處理:

1. 發現任何存取控制相關的異常事件，應立即通報資訊安全主管。
2. 資訊安全主管應根據事件的嚴重性，採取相應的應變措施。
