一、目的

 

本程序旨在建立一套有效率且持續運作的知識管理系統，以收集、儲存、分享及應用組織內部的知識，提升組織效能，促進創新，並確保知識的永續傳承。

二、範圍

本程序適用於所有組織成員，涵蓋所有類型與形式的知識，包括但不限於：

文件 (報告、文件、簡報、表格等)
數據 (統計數據、市場調查、客戶數據等)
流程 (作業流程、標準作業程序等)
專案經驗 (成功案例、失敗經驗、教訓學習等)
專業技能 (技術、知識、經驗等)
隱性知識 (個人經驗、直覺、默會知識等)

三、程序

(一) 知識蒐集:

1. 主動蒐集: 定期透過各種管道蒐集知識，例如：定期舉辦知識分享會、專案結案報告、員工訪談等。
2. 被動蒐集: 建立知識庫，鼓勵員工主動將其擁有的知識貢獻至知識庫。
3. 知識識別: 對蒐集到的知識進行分類、標籤和摘要，方便日後搜尋和利用。

(二) 知識儲存:

1. 知識庫平台: 建立一個集中式的知識庫平台，提供儲存、搜尋、分享和管理知識的功能。
2. 檔案管理: 建立一套完善的檔案命名及歸檔制度，確保知識易於查找及管理。
3. 版本控制: 對知識內容進行版本控制，確保知識的準確性和時效性。
4. 存取權限: 根據不同角色和職責設定不同的存取權限，確保知識的安全性和保密性。

(三) 知識分享:

1. 知識分享平台: 利用知識庫平台或其他工具，促進知識的分享與交流。
2. 知識分享活動: 定期舉辦知識分享會、研討會、培訓課程等活動，促進知識的傳播。
3. 社群平台: 利用內部社群平台或即時通訊軟體，促進知識的即時交流。
4. 導師制度: 建立導師制度，透過經驗傳承的方式分享知識。

(四) 知識應用:

1. 知識搜尋: 提供方便易用的知識搜尋功能，方便員工快速找到所需的知識。
2. 知識應用案例: 收集和分享知識的成功應用案例，鼓勵員工應用知識。
3. 知識回饋機制: 建立知識回饋機制，收集使用者對知識庫的意見和建議，不斷改進知識管理系統。

(五) 知識評估:

1. 定期評估: 定期評估知識管理系統的有效性，並根據評估結果進行調整和改進。
2. 指標追蹤: 追蹤關鍵指標，例如知識庫使用率、知識分享次數等，評估知識管理系統的績效。
3. 使用者滿意度調查: 定期進行使用者滿意度調查，了解員工對知識管理系統的看法和建議。

四、責任

  • 知識管理部門 (或指定人員): 負責制定和維護知識管理程序，管理知識庫平台，並推動知識管理活動。
  • 各部門主管: 負責鼓勵員工參與知識管理，並確保部門內部的知識得到有效管理。
  • 所有員工: 負責將其擁有的知識貢獻至知識庫，並積極參與知識分享和應用。
