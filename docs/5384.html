一、 審查目的：

釐清特定開發行為或計畫區域現有環境狀況，作為環境影響評估（EIA）、環境影響說明書（EIS）或其他環境管理措施規劃的基礎。 目的是確認開發行為可能造成的環境衝擊，並據以擬定減輕措施。

二、 審查對象：

這取決於各地區的法規，可能包括：

  • 開發行為：例如新建廠房、道路工程、都市計畫開發、大型公共建設等。
  • 特定區域：例如自然保護區、水源保護區等需要特別關注環境的區域。
  • 特定活動：例如排放污染物的活動、土地開發利用等。

三、 審查程序步驟 (一般流程)：

1. 申請階段： 開發單位或相關單位向主管機關提出環境現狀審查申請，並提交相關資料，例如：

  • 申請書
  • 計畫書或開發說明
  • 基地位置圖及範圍圖
  • 相關文獻資料
  • 預期環境影響因子說明

2. 審查階段： 主管機關（例如環保單位）會進行審查，可能包括：

  • 文件審查： 審查申請文件是否齊全，內容是否符合規定。
  • 現地調查： 針對申請區域進行現地勘查，收集環境樣本（如水質、空氣、土壤、噪音等），並進行環境監測。
  • 資料分析： 分析收集到的資料，評估現有環境狀況，例如：
    • 生態資源現況（動植物種類、棲息地等）
    • 水文地質環境（水文狀況、地下水位等）
    • 空氣品質
    • 噪音環境
    • 土壤環境
    • 景觀資源

3. 審查結果公告： 主管機關將審查結果公告，並通知申請單位。 結果可能包括：

  • 通過：核發環境現狀審查證明文件。
  • 條件通過：需符合特定條件才能通過。
  • 駁回：申請文件不符規定或環境影響過大。

4. 後續作業： 根據審查結果，開發單位需依據相關規定進行後續作業，例如：編製環境影響評估報告、實施環境監測等。

四、 需提交的資料：

這部分依據審查對象和主管機關要求而異，但一般包含：

  • 地理位置圖和範圍圖：明確開發行為的範圍和位置。
  • 土地利用現況圖：顯示該區域目前的土地使用狀況。
  • 相關環境監測數據：例如水質、空氣品質、噪音、土壤等資料。
  • 生態資源調查資料：例如動植物種類、棲息地等資料。
  • 相關文獻資料：例如既有研究報告、環境規劃資料等。
  • 計畫說明：詳細說明開發計畫的內容及規模。

五、 相關法規：

實際的審查程序和要求會依據各地區的環境保護相關法規而定。 例如台灣地區可能涉及的相關法規包含《環境影響評估法》、《空氣污染防制法》、《水污染防治法》、《噪音管制法》等。 需參考主管機關公布的相關規定。
