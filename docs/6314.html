一、 目的

本指引旨在規範資安健診作業流程，確保健診作業之完整性、一致性及有效性，以提升組織資安防護能力，降低資安風險。

二、 範圍

本指引適用於組織內所有資訊系統、網路設備及相關資安措施之資安健診作業。

三、 健診項目

資安健診項目涵蓋但不限於以下內容：

  • 網路安全:
    • 防火牆設定檢查 (規則審查、日誌分析)
    • 入侵偵測/防禦系統 (IDS/IPS) 設定檢查及效能評估
    • 無線網路安全設定檢查 (WPA2/3加密、訪問控制)
    • VPN 設定及安全性檢查
    • 網路分割及區隔檢查
    • 網路設備安全設定檢查 (Router, Switch等)
    • 漏洞掃描 (針對伺服器、工作站及網路設備)
  • 系統安全:
    • 作業系統安全設定檢查 (修補程式更新、帳號密碼管理、權限控制)
    • 伺服器安全設定檢查 (網頁伺服器、資料庫伺服器、郵件伺服器等)
    • 資料庫安全設定檢查 (帳號密碼管理、權限控制、資料加密)
    • 應用程式安全檢查 (程式碼安全、輸入驗證、輸出編碼)
    • 端點安全 (防毒軟體、個資防護等)
  • 資料安全:
    • 資料分類與分級管理
    • 資料備份與復原機制
    • 資料加密及存取控制
    • 個資保護措施 (符合個資法相關規定)
  • 人員安全:
    • 安全意識教育訓練
    • 身份驗證與授權管理
    • 存取控制政策
  • 物理安全:
    • 機房環境安全 (門禁系統、監視系統、環境監控)
    • 設備安全 (設備鎖定、防盜措施)

四、 作業流程

1. 規劃階段:

  • 確定健診範圍與目標。
  • 制定健診計畫，包含時間表、資源配置及責任分工。
  • 選擇合適的資安工具與技術。

2. 執行階段:

  • 依照計畫執行各項健診項目。
  • 紀錄所有發現的資安弱點及風險。
  • 進行弱點驗證，確認弱點的真實性及嚴重性。

3. 報告階段:

  • 編寫資安健診報告，包含發現的弱點、風險評估及建議的改善措施。
  • 將報告提交相關單位審閱及確認。

4. 改善階段:

  • 根據報告建議，執行必要的改善措施。
  • 驗證改善措施的有效性。

五、 報告內容

資安健診報告應包含以下內容：

  • 健診日期及範圍
  • 健診方法及工具
  • 發現的資安弱點及風險 (包含嚴重性等級)
  • 風險評估結果
  • 建議的改善措施 (包含優先順序及預計完成時間)
  • 附件 (例如：掃描報告、截圖等)

六、 弱點嚴重性等級

  • 建議採用標準的弱點嚴重性等級劃分，例如：高、中、低。

七、 文件管理

  • 所有與資安健診相關的文件應妥善保存及管理。
