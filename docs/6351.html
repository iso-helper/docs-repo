一、 目的

本程序書旨在建立一套完整的資訊安全事故管理流程，以有效地預防、偵測、應變及復原資訊安全事故，降低事故造成的損失，並持續改進資訊安全防護措施。

二、 範圍

本程序書適用於[單位名稱]所有員工及承攬商，涵蓋所有可能影響資訊安全的事故，包括但不限於：

  • 系統入侵
  • 惡意程式感染
  • 數據洩漏
  • 系統故障
  • 網路攻擊
  • 身分盜竊
  • 軟體漏洞
  • 人為錯誤
  • 物理安全事件

三、 責任與權責

  • 資訊安全主管: 負責制定及修訂本程序書，監督事故處理流程，並定期進行稽核。
  • 資訊安全團隊: 負責執行事故應變計畫，提供技術支援及分析。
  • 各部門主管: 負責教育訓練部門員工，並監督員工遵守資訊安全政策及程序。
  • 所有員工: 負責遵守資訊安全政策及程序，並立即通報任何可疑事件。

四、 事故通報程序

1. 發現事故: 任何員工發現任何可能為資訊安全事故的事件，應立即停止相關作業，並立即以電話或電子郵件方式通報資訊安全團隊或主管。
2. 初步評估: 資訊安全團隊接獲通報後，應立即進行初步評估，判斷事故的嚴重程度及影響範圍。
3. 事故記錄: 記錄事故發生的時間、地點、事件描述、影響範圍等相關資訊。 使用標準化的事故報告表格。
4. 通報主管: 資訊安全團隊應將事故通報至相關主管。 嚴重事故需立即通報最高主管。

五、 事故應變程序

1. 隔離受影響系統: 立即隔離受影響的系統或設備，以防止事故擴散。
2. 資料備份及復原: 根據預先規劃的備份及復原計畫，進行資料備份及系統復原。
3. 事故調查: 進行徹底的調查，確定事故原因、影響範圍及損失程度。
4. 修補漏洞: 修補系統漏洞，防止類似事故再次發生。
5. 預防措施: 根據事故調查結果，採取必要的預防措施，以降低未來發生類似事故的風險。
6. 事後報告: 完成事故應變後，編寫事故報告，說明事故過程、處理方法及結果，並提出改進建議。

六、 事故分類與嚴重程度

事故嚴重程度將依據其影響範圍、損失程度及影響時間等因素進行評估，並分為以下等級：

  • 低級: 影響有限，可輕易修復，不影響業務運作。
  • 中級: 影響範圍較廣，需要一定時間修復，可能影響部分業務運作。
  • 高級: 影響範圍極廣，需要長時間修復，嚴重影響業務運作，可能造成重大損失。

七、 事故報告及稽核

1. 每一起事故均需編寫事故報告，並提交至資訊安全主管審核。
2. 資訊安全主管應定期稽核本程序書的執行情況，並根據稽核結果進行必要的修訂。
