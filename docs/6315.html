1. 目的

本程序書旨在規範資訊安全組織的管理，確立組織架構、職責、權限及運作流程，以有效執行資訊安全政策，保障組織資產安全。

2. 適用範圍

本程序書適用於所有參與資訊安全管理的人員，包括但不限於資訊安全部門、各部門資訊安全管理人員、系統管理員及所有員工。

3. 組織架構

(請根據實際情況填寫，例如圖表或文字說明)

  • 資訊安全主管: 負責資訊安全策略規劃、執行及監督。
  • 資訊安全部門: 負責資訊安全技術的實施、維護及監控。 (可細分為子部門，例如：安全架構、安全監控、事件回應等)
  • 各部門資訊安全管理人員: 負責其部門資訊安全政策的執行及落實。

4. 職責與權限

(請根據實際情況填寫，可使用表格呈現)

| 角色 | 職責 | 權限 |
|-----------------|-------------------------------------------------------------------|-----------------------------------------------------------------|
| 資訊安全主管 | 制定並執行資訊安全策略、管理資訊安全預算、監督安全事件處理 | 批准安全相關預算、任免資訊安全部門人員、審核安全事件報告 |
| 資訊安全部門 | 實施及維護資訊安全技術、監控安全事件、提供安全培訓 | 執行安全措施、審查系統安全性、進行安全漏洞評估 |
| 各部門資訊安全管理人員 | 執行部門資訊安全政策、報告安全事件、配合資訊安全部門工作 | 執行部門內部安全控制措施，向資訊安全主管報告安全事件 |
| 所有員工 | 遵守資訊安全政策、保護公司資訊資產、報告可疑活動 | 存取其授權的資訊系統及資料 |

5. 作業流程

(以下列出一些常見的流程，請根據實際需求增減或修改)

5.1 安全政策制定與修訂:

1. 資訊安全主管根據組織需求及法規，制定或修訂資訊安全政策。
2. 政策草案需經相關部門審核。
3. 政策經主管批准後發布實施。
4. 定期評估政策有效性並進行修訂。

5.2 安全事件處理:

1. 事件發現者立即向資訊安全部門報告。
2. 資訊安全部門根據事件嚴重程度，啟動應變計畫。
3. 事件處理完成後，需撰寫事件報告，並進行後續改善措施。

5.3 安全培訓:

1. 資訊安全部門定期提供安全培訓。
2. 培訓內容涵蓋資訊安全政策、安全操作規範、常見威脅及防範措施等。
3. 培訓記錄需妥善保存。

5.4 資產管理:

1. 所有資訊資產需進行編號及分類管理。
2. 定期進行資產盤點，確保資產安全。
3. 制定資產處置流程。

5.5 風險管理:

1. 定期進行風險評估，識別潛在風險。
2. 制定風險應對策略，降低風險。
3. 追蹤風險，並定期檢討風險應對措施。

6. 文件管理

  • 所有相關文件，包含本程序書、安全政策、事件報告等，需妥善保存並定期更新。
