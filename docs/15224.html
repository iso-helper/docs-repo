目的: 建立一套標準化的文件管制程序，以確保所有文件的完整性、準確性、及可追蹤性，並有效管理文件的生命週期。

適用範圍: 本程序適用於所有公司內部產生及使用的文件，包括但不限於：文件、表格、圖表、電子郵件、電子檔案等。

責任:

  • 文件發起人: 負責文件的撰寫、審核及提交。
  • 主管/審核人: 負責文件的審核及批准。
  • 文件管理員: 負責文件的登錄、歸檔、版本控制及銷毀。

程序:

1. 文件創建與審核:

  • 文件發起人根據公司既定格式及規範撰寫文件。
  • 文件需經相關主管或審核人審核通過，並簽名或電子簽章確認。
  • 審核內容包含：內容正確性、完整性、格式一致性、符合法規等。
  • 審核意見需記錄在文件上或相關系統中。

2. 文件發佈與版本控制:

  • 審核通過的文件將由文件管理員發佈，並賦予唯一的版本號碼及發佈日期。
  • 所有發佈的文件應儲存在指定的儲存位置，例如文件管理系統或檔案伺服器。
  • 版本控制需明確記錄所有版本的文件內容、修改日期、修改人及修改原因。
  • 新版本發佈後，舊版本應保留至規定的時間，並標記為「已過期」。

3. 文件存取與使用:

  • 文件存取權限應根據職責劃分，並進行嚴格控制。
  • 授權人員才能存取和使用相關文件。
  • 文件的使用應符合公司的保密規定。

4. 文件維護與更新:

  • 文件應定期進行檢討和更新，以確保其內容的準確性和時效性。
  • 文件更新需遵循文件創建與審核程序。
  • 更新後的版本需明確標示版本號碼及更新日期。

5. 文件銷毀:

  • 過期或作廢的文件應依據公司的檔案管理規定進行銷毀。
  • 銷毀需有記錄，包含銷毀日期、銷毀方式及銷毀人。
  • 銷毀方式應確保文件內容無法恢復。
