一、 目的

本程序旨在建立一套有效的知識管理機制，促進組織內知識的創造、儲存、分享和應用，提升組織學習能力和競爭力，並確保知識的持續性與可用性。

二、 適用範圍

本程序適用於組織內所有人員，涵蓋所有與知識相關的活動，包括但不限於文件、數據、經驗、技能和專案知識。

三、 關鍵名詞定義

  • 知識: 指經過組織和個人經驗累積，能提升組織效率和創造價值的信息和見解。
  • 知識創造: 指透過學習、研究、實踐等活動產生新的知識。
  • 知識儲存: 指將知識以有效的方式記錄、歸檔和保存。
  • 知識分享: 指將知識傳遞給組織內其他人員。
  • 知識應用: 指將知識應用於實際工作中，以提升效率和解決問題。
  • 知識管理系統 (KMS): 指用於支持知識管理流程的資訊系統，例如文件管理系統、知識庫、社群平台等。

四、 程序流程

(一) 知識創造:

1. 鼓勵學習: 透過培訓、研討會、工作坊等方式，促進員工學習和技能提升。
2. 經驗分享: 鼓勵員工分享工作經驗和心得，例如撰寫經驗報告、參與知識分享會議等。
3. 問題解決: 鼓勵員工積極解決問題，並將解決方案記錄下來，供日後參考。
4. 創新思維: 鼓勵員工提出創新想法，並提供平台進行驗證和應用。

(二) 知識儲存:

1. 標準化流程: 建立標準化的文件管理流程和命名規範，確保知識易於查找和存取。
2. 知識庫建置: 建立組織內部的知識庫，將重要的知識、文件和資訊集中儲存。
3. 版本控制: 實施版本控制機制，確保知識的完整性和一致性。
4. 資訊安全: 確保知識的安全性，防止知識外洩或損失。

(三) 知識分享:

1. 知識分享平台: 建立線上或線下的知識分享平台，例如內部社群、知識庫、定期會議等。
2. 知識社群: 鼓勵員工參與知識社群，互相學習和交流。
3. 專家諮詢: 提供專家諮詢服務，協助員工解決疑難雜症。
4. 導師制度: 建立導師制度，促進經驗傳承。

(四) 知識應用:

1. 知識搜尋: 提供方便的知識搜尋工具，讓員工快速找到所需的知識。
2. 知識應用案例: 分享知識應用案例，鼓勵員工學習和應用。
3. 績效考核: 將知識應用納入績效考核指標，鼓勵員工積極應用知識。
4. 持續改善: 持續檢討和改善知識管理流程，提升知識管理效率。

五、 責任分工

  • 部門主管: 負責推動部門內的知識管理工作，確保員工參與知識創造、儲存、分享和應用。
  • 知識管理負責人: 負責規劃和執行組織的知識管理策略，並維護知識管理系統。
  • 所有員工: 有責任創造、儲存、分享和應用知識。

六、 程序修訂

  • 本程序將定期檢討和修訂，以適應組織的發展和需求。
