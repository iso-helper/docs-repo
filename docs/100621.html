1. 目的

本程序書旨在建立一套系統化的流程，用於識別、分析、糾正不符合 ISO 55001 標準要求的事件（糾正措施），以及預防此類事件再次發生（預防措施）。此程序書適用於[公司名稱]的所有資產管理活動。

2. 範圍

本程序書適用於[公司名稱]所有部門和員工，涵蓋所有與資產管理相關的活動，包括但不限於：資產規劃、採購、營運、維護、處置等。

3. 責任

  • 資產管理負責人: 負責監督整個糾正和預防措施程序的實施和有效性。
  • 部門主管: 負責在其部門內執行本程序書，並確保員工理解並遵守相關規定。
  • 相關員工: 負責識別、報告並參與糾正和預防措施的制定和實施。

4. 程序

4.1 糾正措施

1. 不符合項的識別: 任何不符合 ISO 55001 要求的事件（例如：資產故障、維護延誤、成本超支、績效指標未達標等）都應被立即識別並記錄。 可以使用檢查表、事件報告表等工具進行記錄。

2. 不符合項的分析: 對不符合項進行根本原因分析 (RCA)，例如使用魚骨圖、5W2H 等方法，以確定根本原因。 分析應包含事件的描述、影響、根本原因及證據。

3. 糾正措施的制定: 根據根本原因分析結果，制定有效的糾正措施，以消除不符合項及其根本原因。 糾正措施應包括具體的行動、負責人、時間表和資源分配。

4. 糾正措施的實施: 按照制定的計劃實施糾正措施。

5. 糾正措施的有效性驗證: 在實施糾正措施後，應驗證其有效性，確保不符合項已得到有效解決，並防止再次發生。

6. 記錄保存: 所有與糾正措施相關的記錄（包括不符合項報告、根本原因分析報告、糾正措施計劃、實施記錄和有效性驗證記錄）都應妥善保存。

4.2 預防措施

1. 潛在問題的識別: 積極主動地識別潛在的風險和問題，例如通過風險評估、管理審查、員工回饋等方式。

2. 預防措施的制定: 針對識別出的潛在問題，制定有效的預防措施，以防止不符合項的發生。

3. 預防措施的實施: 按照制定的計劃實施預防措施。

4. 預防措施的有效性驗證: 定期評估預防措施的有效性，並根據需要進行調整。

5. 記錄保存: 所有與預防措施相關的記錄（包括風險評估報告、預防措施計劃、實施記錄和有效性驗證記錄）都應妥善保存。
